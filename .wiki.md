# Project Summary
The Healthcare Dashboard is a comprehensive MERN stack application designed to enhance patient health management. It provides features such as medication tracking, wellness activities, and real-time data monitoring, fostering collaboration between patients and caregivers, particularly through Telehealth services. The platform operates on a subscription model, offering both free and premium plans, and prioritizes user experience and data security.

# Project Module Description
- **Patient Profile**: Manages user information and authentication.
- **Vital Log**: CRUD operations for tracking vital signs.
- **Medication Management**: Schedules and adherence tracking for medications.
- **Device Integration**: Captures data from connected health devices.
- **Data Visualization**: Personalized health analytics dashboard.
- **Care Plans**: Task management and predictive health insights.
- **Wellness Guide**: Resources for mental health and nutrition.
- **Telehealth**: Appointment booking and access to medical records.
- **Care Team Management**: Manages caregiver access.
- **Health Education Center**: Provides educational resources.
- **Settings & Preferences**: User profile and privacy settings.
- **Security Center**: Monitors account security and enables two-factor authentication.
- **Authentication**: Facilitates secure login and registration.
- **Subscription Management**: Oversees user subscriptions.
- **Health Dashboard**: Provides an overview of health metrics and appointments.
- **Medication Request**: Allows digital prescription submissions.
- **Video Modal**: Displays educational videos.
- **API Integration**: Handles custom API requests using native fetch.

# File Description Inventory
- **backend/.env**: Configuration for environment variables.
- **backend/config/db.js**: MongoDB connection configuration.
- **backend/models/**: Schemas for User, Vital, Medication, and Device.
- **backend/routes/auth.js**: User authentication management.
- **backend/routes/**: API route files for user management and health data.
- **backend/middleware/auth.js**: Middleware for validating JWT tokens.
- **backend/scripts/seed.js**: Script for populating the database with sample data.
- **backend/server.js**: Main application logic and API server setup using Express.js.
- **frontend/api.js**: Centralized API management with automatic token handling and error management.
- **frontend/lib/api.ts**: Handles API integration with the backend.
- **frontend/hooks/useSupabaseData.tsx**: Custom hook for fetching user data from the backend API.
- **frontend/pages/AuthPage.tsx**: Authentication page with direct API call functionality.

# Technology Stack
- **React**: Frontend library for building user interfaces.
- **TypeScript**: For type safety.
- **Tailwind CSS**: Utility-first CSS framework.
- **Vite**: Fast development build tool.
- **Node.js & Express**: Backend server framework.
- **MongoDB**: NoSQL database for data storage.
- **JWT & Bcrypt**: Secure authentication.
- **Mongoose**: ODM for MongoDB.
- **Morgan**: HTTP request logger middleware.
- **Native Fetch API**: For API requests.

# Usage
1. Install backend dependencies:
   ```bash
   cd backend
   npm install
   ```
2. Set up environment variables in `.env`.
3. Run the backend server:
   ```bash
   npm start
   ```
4. For the frontend, install dependencies:
   ```bash
   cd frontend
   npm install
   ```
5. Run the frontend application:
   ```bash
   npm run dev
   ```
6. To seed the database with sample data, run:
   ```bash
   cd backend
   node scripts/seed.js
   ```
